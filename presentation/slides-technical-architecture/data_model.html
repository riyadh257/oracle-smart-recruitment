Data Model & Schema
Core Entities
Users
Authentication and role management (admin, employer, candidate)
Candidates
Profile data with skills, experience, preferences, and AI-inferred attributes
Employers
Company profiles with culture attributes and operational metrics
Jobs
Job postings with enriched descriptions and ideal candidate profiles
Applications
Candidate-job matches with AI scoring and billing qualification
Interviews
Scheduling, feedback collection, and QR code access
Relationships: Users → Candidates/Employers (1:1) | Employers → Jobs (1:N) | Candidates + Jobs → Applications (M:N) | Applications → Interviews (1:N)
Schema Features
JSON Flexibility
Flexible attribute storage using JSON fields enables 10,000+ attribute matching:
technicalSkills softSkills workStyleAttributes personalityTraits cultureFitPreferences aiInferredAttributes
AI-Powered Enrichment
AI inference fields store machine learning outputs for enhanced matching accuracy and profile completeness scoring.
KSA Compliance
Dedicated tables for Nitaqat tracking, MHRSD sync, and Qiwa integration ensure regulatory compliance.
