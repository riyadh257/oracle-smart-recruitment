Data Flow: AI Analysis Pipeline
1
File Upload to S3
User uploads resume/video file via frontend → tRPC mutation → S3 storage with unique key generation
storagePut() S3 Bucket Public URL
2
Backend File Retrieval
Backend downloads file from S3 URL → validates file type and size → prepares for AI processing
File Validation Size Check MIME Type
3
AI API Invocation
Call appropriate AI service: invokeLLM() for resume parsing or transcribeAudio() for video interviews
LLM API Whisper API Retry Logic
4
Response Processing
Parse AI response → extract structured data (skills, experience, sentiment) → validate output schema
JSON Schema Data Validation Error Handling
5
Structured Storage
Store analysis results in database → update candidate profile with extracted insights → cache results
MySQL Storage Redis Cache JSON Fields
6
Insights & Ranking
Return insights to frontend → feed into candidate ranking algorithms → update application scores
Ranking Engine Score Calculation Real-time Updates
AI Services Integration
LLM API (Resume Parsing)
Extracts skills, experience, education from resume text using structured JSON schema output
Whisper API (Transcription)
Converts video interview audio to text with timestamp segments and language detection
Performance Optimization
Response Caching
Redis cache with 24h TTL for identical requests
Retry Strategy
Exponential backoff with 3 retry attempts
Timeout Handling
30s timeout with graceful degradation
Async Processing
Background jobs for large file analysis
