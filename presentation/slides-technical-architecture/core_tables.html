Core Database Tables
users
  • id PK
  • openId (OAuth)
  • email, name
  • role (admin/employer/candidate)
candidates
  • id PK
  • userId FK
  • fullName, email, phone
  • technicalSkills (JSON)
  • aiInferredAttributes (JSON)
employers
  • id PK
  • userId FK
  • companyName, industry
  • cultureAttributes (JSON)
  • billingModel, accountStatus
jobs
  • id PK
  • employerId FK
  • title, location, workSetting
  • requiredSkills (JSON)
  • aiInferredRequirements (JSON)
applications
  • id PK
  • candidateId, jobId FK
  • overallMatchScore (0-100)
  • matchBreakdown (JSON)
  • status, qualifiesForBilling
